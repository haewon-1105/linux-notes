# Day 4 - Linux Troubleshooting & Security Practice

## What I Learned

Today was focused on hands-on Linux administration scenarios rather than learning completely new commands. I applied previously learned Linux commands to solve real-world system administration, troubleshooting, and security challenges.

---

## Scenario 1: Project Organization

In this exercise, I acted as a system administrator responsible for organizing a development environment for a software project.

### Tasks Completed

* Created a logical project directory structure using `mkdir`
* Moved and organized files using `mv`
* Created backup copies using `cp`
* Archived project files using `tar`
* Removed unnecessary files using `rm`

### Key Takeaway

I learned how Linux administrators organize project environments, protect important files with backups, and maintain a clean file structure.

---

## Scenario 2: Log Investigation & Troubleshooting

In this exercise, I investigated system and application issues by analyzing log files and configuration differences.

### Tasks Completed

* Searched log files using `grep`
* Compared configuration files using `diff`
* Inspected system messages using `dmesg`
* Combined commands using pipes (`|`)
* Saved investigation results using output redirection (`>`)

### Key Takeaway

I learned how Linux administrators identify errors, investigate system problems, and collect evidence for troubleshooting.

---

## Scenario 3: Linux Security & Permissions

In this exercise, I configured file and directory permissions to create a secure collaboration environment.

### Tasks Completed

* Managed file permissions with `chmod`
* Changed file ownership with `chown`
* Configured secure directory access
* Applied `setgid` to enable shared group ownership in collaborative workspaces

### Key Takeaway

I learned how Linux permissions and ownership protect sensitive data while allowing teams to collaborate securely.

---

## Skills Practiced

* File and directory management
* Backup and archiving
* Log analysis
* Troubleshooting methodology
* Linux permissions and ownership
* Team collaboration security
* Real-world Linux administration workflows

---

## Reflection

Today's exercises showed how Linux commands are used together in real administration tasks. Instead of learning individual commands, I practiced applying them to realistic scenarios involving project organization, troubleshooting, and security management.
